第八章：学习站——记录成长轨迹

本章目标：实现一个"学习站"功能，记录正在学习的技术、学习进度、笔记和资源链接。

学完后你能做什么：有一个个人学习管理面板，可以添加学习项目、标记进度、写学习笔记。你能理解 CRUD（增删改查）的完整实现。

8.1 为什么要有"学习站"？

很多开发者都有"学了很多但忘了"的困扰。今天看了 React 文档，下周就忘了 useEffect 的依赖数组怎么用。学习站的作用就是：把学习过程"外化"——记录下来，随时回顾。

8.1.1 费曼学习法

物理学家理查德·费曼有一个著名的学习方法："如果你不能用简单的语言解释一个概念，说明你还没真正理解它。"

费曼学习法的四个步骤：

  1. 选择一个概念
  2. 假装教给一个新手（用简单语言）
  3. 卡壳时回去重新学习
  4. 简化和类比

学习站就是这个过程的"记录器"。你每学一个技术，都写一篇笔记——这既帮助理解，也留下了一个可检索的知识库。

8.1.2 学习站的核心功能

  • 学习项目管理：添加正在学习的技术（如 React、Docker、Python）
  • 进度追踪：用百分比表示学习进度（0%~100%）
  • 状态标记：正在学 / 已完成 / 暂停
  • 学习笔记：记录关键知识点、遇到的问题、解决方案
  • 资源链接：收藏相关的教程、文档、视频链接

📖 故事时间：Anki 与间隔重复

学习的一个核心问题是"遗忘曲线"——学完后如果不复习，几天内就会忘掉大部分内容。德国心理学家赫尔曼·艾宾浩斯在 1885 年发现了这个现象，并绘制了著名的"艾宾浩斯遗忘曲线"。

2006 年，大学生 Damien Elmes 开发了一款开源软件 Anki，基于"间隔重复"（Spaced Repetition）算法。它的原理是：在遗忘即将发生的时间点复习，用最少的复习次数达到最好的记忆效果。

Anki 的算法会在你第一次学习后 1 天让你复习，然后是 3 天、7 天、15 天、30 天……随着记忆越来越牢固，间隔越来越长。今天 Anki 有数百万用户，是语言学习者和医学生的必备工具。

我们的学习站虽然没有 Anki 那么复杂的算法，但"记录+回顾"的核心思想是一致的。

8.2 数据库设计——learning_items 表

CREATE TABLE IF NOT EXISTS learning_items (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  category TEXT,
  status TEXT DEFAULT 'learning',
  progress INTEGER DEFAULT 0,
  notes TEXT,
  resource_url TEXT,
  user_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

字段说明：

字段类型说明
titleTEXT学习项目名称，如 "React Hooks 深入"
categoryTEXT分类，如 "前端"、"后端"、"DevOps"
statusTEXTlearning / completed / paused
progressINTEGER进度百分比（0-100）
notesTEXT学习笔记
resource_urlTEXT资源链接
user_idINTEGER用户ID，外键

status 字段有三个取值：

  • learning：正在学（默认）
  • completed：已完成
  • paused：暂停

🤔 为什么 progress 和 status 都要存？

你可能觉得这两个字段冗余——progress=100 不就是 completed 吗？为什么还要 status？

问题在于：progress=100 不一定意味着"完成了"。用户可能只是把课程视频全部看了一遍（progress=100），但还没有做练习、没有复习。status 表示用户的主观状态，progress 表示客观进度。两者结合才能准确描述学习状态。

另一个场景：用户学到 80% 时暂停了（status=paused, progress=80）。如果只存 progress，就不知道用户是"正在学 80%"还是"暂停在 80%".

8.3 后端 API——完整的 CRUD

CRUD 是数据库操作的四个基本操作：Create（创建）、Read（读取）、Update（更新）、Delete（删除）。

8.3.1 获取学习列表（Read）

app.get('/api/learning', (req, res) => {
  if (!req.session.userId) {
    return res.status(401).json({ error: '请先登录' });
  }
  
  db.all(
    'SELECT * FROM learning_items WHERE user_id = ? ORDER BY created_at DESC',
    [req.session.userId],
    (err, rows) => {
      if (err) return res.status(500).json({ error: err.message });
      res.json(rows);
    }
  );
});

8.3.2 添加学习项目（Create）

app.post('/api/learning', (req, res) => {
  if (!req.session.userId) {
    return res.status(401).json({ error: '请先登录' });
  }
  
  const { title, category, resource_url } = req.body;
  
  if (!title) {
    return res.status(400).json({ error: '标题不能为空' });
  }
  
  db.run(
    'INSERT INTO learning_items (title, category, resource_url, user_id) VALUES (?, ?, ?, ?)',
    [title, category || '', resource_url || '', req.session.userId],
    function(err) {
      if (err) return res.status(500).json({ error: err.message });
      res.json({ id: this.lastID, message: '添加成功' });
    }
  );
});

8.3.3 更新进度（Update）

app.put('/api/learning/:id', (req, res) => {
  if (!req.session.userId) {
    return res.status(401).json({ error: '请先登录' });
  }
  
  const { progress, status, notes } = req.body;
  const itemId = req.params.id;
  
  // 验证 progress 范围
  if (progress !== undefined && (progress < 0 || progress > 100)) {
    return res.status(400).json({ error: '进度必须在 0-100 之间' });
  }
  
  db.run(
    'UPDATE learning_items SET progress = ?, status = ?, notes = ? WHERE id = ? AND user_id = ?',
    [progress, status, notes, itemId, req.session.userId],
    function(err) {
      if (err) return res.status(500).json({ error: err.message });
      if (this.changes === 0) {
        return res.status(403).json({ error: '只能修改自己的学习项目' });
      }
      res.json({ message: '更新成功' });
    }
  );
});

this.changes === 0 的判断很重要：它表示 UPDATE 没有影响任何行。这发生在两种情况下：1）itemId 不存在；2）存在但不属于当前用户。我们统一返回 403，不区分具体原因（信息隐藏原则）。

8.3.4 删除（Delete）

app.delete('/api/learning/:id', (req, res) => {
  if (!req.session.userId) {
    return res.status(401).json({ error: '请先登录' });
  }
  
  db.run(
    'DELETE FROM learning_items WHERE id = ? AND user_id = ?',
    [req.params.id, req.session.userId],
    function(err) {
      if (err) return res.status(500).json({ error: err.message });
      if (this.changes === 0) {
        return res.status(403).json({ error: '只能删除自己的学习项目' });
      }
      res.json({ message: '删除成功' });
    }
  );
});

8.4 前端页面

8.4.1 进度条组件

进度条是 CSS 的经典应用——一个外层容器作为背景，一个内层元素根据进度设置宽度：

/* 进度条样式 */
.progress-bar {
  width: 100%;
  height: 8px;
  background: var(--bg-secondary);
  border-radius: 4px;
  overflow: hidden;
}

.progress-fill {
  height: 100%;
  background: var(--accent-gold);
  border-radius: 4px;
  transition: width 0.3s ease;
}

/* 根据进度动态设置 width */
.progress-fill[data-progress="30"] { width: 30%; }
.progress-fill[data-progress="60"] { width: 60%; }
.progress-fill[data-progress="100"] { width: 100%; }

实际使用时，用 JavaScript 动态设置宽度：

function renderProgress(percent) {
  return `
    <div class="progress-bar">
      <div class="progress-fill" style="width: ${percent}%"></div>
    </div>
    <span class="progress-text">${percent}%</span>
  `;
}

8.5 知识点回顾

📚 本章核心概念回顾

  • CRUD：Create（创建）、Read（读取）、Update（更新）、Delete（删除）。数据库操作的四个基本操作。
  • progress + status：客观进度和主观状态结合，准确描述学习情况。
  • this.changes：SQLite 回调中 this.changes 表示影响的行数。0 表示没有匹配的行。
  • 信息隐藏：更新/删除失败时统一返回 403，不区分"不存在"和"无权"。
  • CSS 进度条：外层容器 + 内层填充元素，width 根据进度动态设置。
  • range 滑块：HTML5 input[type="range"] 用 CSS 自定义样式。

8.6 常见问题急救站

问题现象原因解决方案
进度不更新 拖动滑块后端没变化 PUT 请求 body 格式错误或没发出去 检查 fetch 的 method 和 body；确保 Content-Type: application/json
进度条宽度不对 进度 50% 但进度条只占 10% 没把 progress 值转成百分比宽度 style="width: ${item.progress}%"
只能看自己的 别人的学习项目看不到 API 过滤了 user_id 这是设计如此，如果要公开需要改 API 去掉 user_id 过滤
this.changes 未定义 更新成功但 this.changes 报错 使用了箭头函数，this 不指向正确的对象 用普通 function 而不是箭头函数作为回调
进度超过 100 滑块拖到 150% 前端没限制最大值 input max="100"；后端也验证 0-100

A.1 HTTP 状态码速查

状态码含义常见场景
200 OK请求成功GET/POST 正常返回
201 Created创建成功POST 创建资源后返回
400 Bad Request请求参数错误缺少必填字段
401 Unauthorized未认证没登录就访问需要登录的接口
403 Forbidden无权限登录了但不是管理员
404 Not Found资源不存在URL 错误或数据不存在
409 Conflict冲突用户名已存在
500 Internal Server Error服务器内部错误代码抛异常、数据库连接失败

A.2 SQL 速查

A.2.1 数据定义（DDL）

-- 创建表
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 修改表（添加列）
ALTER TABLE users ADD COLUMN email TEXT;

-- 删除表
DROP TABLE users;

-- 创建索引（加速查询）
CREATE INDEX idx_username ON users(username);

A.2.2 数据操作（DML）

-- 插入
INSERT INTO users (username) VALUES ('霁光');

-- 查询
SELECT * FROM users;
SELECT username, created_at FROM users WHERE id = 1;
SELECT * FROM users ORDER BY created_at DESC LIMIT 10;

-- 更新
UPDATE users SET username = '新名字' WHERE id = 1;

-- 删除
DELETE FROM users WHERE id = 1;

A.2.3 联表查询

-- INNER JOIN（只返回匹配的行）
SELECT posts.*, users.username
FROM posts
JOIN users ON posts.user_id = users.id;

-- LEFT JOIN（返回左表所有行）
SELECT posts.*, users.username
FROM posts
LEFT JOIN users ON posts.user_id = users.id;

-- 聚合
SELECT user_id, COUNT(*) as count FROM posts GROUP BY user_id;

A.8 扩展方向推荐

完成本书后，你可以继续深入以下方向：

A.8.1 前端进阶

  • Vue.js / React：学习现代前端框架，提升开发效率
  • TypeScript：为 JavaScript 添加类型系统，减少运行时错误
  • Tailwind CSS：原子化 CSS 框架，快速构建界面
  • Next.js / Nuxt.js：全栈框架，支持 SSR、SSG、API 路由

A.8.2 后端进阶

  • RESTful API 设计：学习 API 设计的最佳实践
  • JWT 认证：用 JSON Web Token 替代 Session
  • Redis：内存数据库，做缓存和消息队列
  • Docker：容器化部署，环境一致性
  • GraphQL：灵活的数据查询语言

A.8.3 数据库进阶

  • MySQL / PostgreSQL：学习大型关系数据库
  • 索引优化：理解 B+ 树索引，优化查询性能
  • 事务：ACID 特性，并发控制
  • MongoDB：NoSQL 文档数据库

A.8.4 运维进阶

  • CI/CD：GitHub Actions 自动测试和部署
  • 监控：日志收集、性能监控、告警
  • CDN：内容分发网络，加速静态资源
  • Kubernetes：容器编排，大规模部署
